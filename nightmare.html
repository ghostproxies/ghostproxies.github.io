Case Study 1

A client requires an efficient non-cryptographic PRNG in C# for 64-bit architecture, 8-bit output, a minimum period of at least 2⁶⁴ and reversibility.

The client application requires the fastest PRNG speed possible without failing 1TB stdin8 PractRand 0.96 tests.

GhostProxies delivers MatterBurst™ and satisfies client requirements to increase throughput in a memory-constrained queue system that shuffles tiny arrays.

public class MatterBurst { public ulong a; public ulong b; public byte Next() { a += 1111111111; b += (b >> 8) + a; return (byte)b; } }

The Next function modifies the state in a MatterBurst instance to generate a pseudorandom byte integer as the return value.

Each state variable (a and b) in each MatterBurst instance must be seeded.

Case Study 2

A client requires an efficient non-cryptographic PRNG in C for 64-bit architecture, 1024-bit output, PulseField™, a minimum period of at least 2⁶⁴ and reversibility.

The client application requires PRNG speed that's faster than shishua (the fastest PRNG in the world) without failing each NIST STS test within Dieharder 3.31.1.

GhostProxies delivers IonGush™ and satisfies client requirements to increase frame rates in a 3D engine that uses advanced graphics rendering algorithms.

#include <stdint.h> struct iongush_state { uint64_t output[16]; uint64_t a; uint64_t b; uint64_t c; }; void iongush(struct iongush_state *s) { char i = 0; s->a = ((s->a << 27) | (s->a >> 37)) ^ s->b; s->b += 11111111111111111; s->c = s->a; while (i < 16) { s->output[i] = (s->c << 19) | (s->c >> 45); s->output[i + 1] = s->output[i] + s->c; s->c = ((s->c << 41) | (s->c >> 23)) + s->a; s->output[i + 2] = s->b + s->c; s->output[i + 3] = s->output[i] ^ s->c; i += 4; } }

The iongush function modifies the state in a struct iongush_state instance to generate 16 pseudorandom uint64_t integers in the output array.

Each state variable (a, b, c and each integer in the output array) in each struct iongush_state instance must be seeded.

It requires the stdint.h header to define a 64-bit, unsigned integral type for uint64_t.

Case Study 3

A client requires an efficient search algorithm in Go for the first occurrence of a needle within a haystack array of elements that have an unknown distribution.

The client application requires haystack to contain a fixed range of 30 to 40 elements that are already sorted in ascending order before searching.

GhostProxies delivers ChargeSearch™ and satisfies client requirements to decrease round-trip latency when repeatedly searching tiny sorted arrays of 4-byte tokens.

func ChargeSearch(haystack []uint32, needle uint32) int { low := 0 high := len(haystack) - 1 if haystack[high] >= needle { if haystack[0] < needle { high >>= 1 if haystack[high] < needle { low = high } high >>= 2 if haystack[low + high] < needle { low += high if haystack[low + high] < needle { low += high if haystack[low + high] < needle { low += high if haystack[low + high] < needle { low += high } } } } for haystack[low] < needle { low++ } } if haystack[low] == needle { return low } } return -1 }
